# Publish Checklist

Usar esta lista en la ultima revision antes de subir el proyecto.

## Repo

- confirmar que el repo solo tenga archivos publicables
- revisar `.gitignore`
- no subir logs, caches, temporales ni entornos virtuales
- confirmar que `FINANZAS_demo.xlsx` y `mercado_pago_demo.xlsx` sean los archivos demo correctos

## README

- explicar que hace la app en pocas lineas
- mostrar fuentes de entrada y salida
- dejar clara la parte de demo y privacidad
- revisar ortografia y nombres de secciones

## App

- probar `Archivo Excel`
- probar `Captura bancaria`
- revisar preview, importacion e insights
- confirmar que no haya textos raros, nombres inconsistentes o botones confusos

## Capturas

- guardar las imagenes en `docs/screenshots/`
- seguir `docs/demo-capture-plan.md`
- revisar `docs/privacy-checklist.md`
- confirmar que todas tengan mismo criterio visual

## Antes de publicar

- abrir el repo como si fueras un tercero
- revisar que todo se entienda sin explicacion extra
- si algo genera duda, simplificarlo antes de subir
